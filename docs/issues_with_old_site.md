- too many duplicate books. We store alternate_titles as an array text field on books which is used in search to find dupes
- zero handling of "series". The Lord of the Rings is really 3 books, but it's almost always on lists as just "The Lord of the Rings". The only site has a series field on book, but it's just plain text and populated by AI, and it's not exposed. It's basically garbage. We also need to handle stuff like annotated books, illustrationed version of books. they are technically associated with the parent "work" and have their own ISBNs. we should capture this.
- multi book groupings. since we allow users to import their own books (from goodreads) there's a lot of garbage on the site and in search results. Like a combined book that contains muliple books: "Of Mice And Men / Cannery Row", "The Pearl/The Red Pony", "Heart Of Darkness & The Secret Sharer". We need a better way to handle these. we should know that these style of books are associated with multiple books and should be linked to the books. 
- bad handling of author duplicates. W. E. B. Du Bois, William Edward Burghardt Du Bois. Richard Bachman vs Stephen King. We do use a name parsing library that attempts to parse out last name, initials, first name, etc but it doesn't work the best and think it could be improved.
